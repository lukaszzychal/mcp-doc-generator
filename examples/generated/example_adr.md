# ADR-{{number}}: ADR-001: Migration from REST to GraphQL

**Data:** 2025-01-15  
**Status:** Accepted  
**Autor:** Architecture Team

## Kontekst

Current REST API has over-fetching and under-fetching issues. Multiple endpoints make mobile app slow (too many requests). Frontend teams want more flexibility in data queries. Need better real-time capabilities.

## Decyzja

Migrate to GraphQL API while maintaining REST for backwards compatibility during transition period.
